 1. Backend & Forensic Engine (Python 3.x)
  The core "brain" of the tool is written in Python, chosen for its powerful binary manipulation and cybersecurity
  libraries.


   * FastAPI: A high-performance web framework used to build the REST API. It handles file uploads, analysis
     requests, and serves the reports.
   * Uvicorn: An ASGI web server that runs the FastAPI application.
   * Scapy: A powerful packet manipulation library used for the Network Analyzer. It reassembles TCP streams and     
     detects ICMP/DNS steganography.
   * Pillow (PIL): Used for the Image Analyzer. It handles metadata extraction, pixel-level LSB (Least Significant   
     Bit) analysis, and visual noise detection.
   * pypdf: Used for the PDF Analyzer to inspect internal object streams, detect hidden attachments, and find data   
     appended after the EOF.
   * Cryptography: A security library used for the Crypto Manager. It handles high-grade encryption/decryption       
     (AES-256-GCM, ChaCha20, and RSA).
   * Filetype: Used for the File Identifier to detect the true type of a file based on its "magic bytes" rather than 
     just its extension.
   * Colorama: Used in the CLI to provide color-coded terminal output (Red for alerts, Green for clean).

  2. Core Forensic Logic
  The engine uses several specialized modules to find hidden data:


   * Recursive Engine: A custom orchestrator that dives into "nested" artifacts. For example, if it finds a ZIP
     inside an Image, it extracts the ZIP and starts a new analysis on the files inside.
   * YaraScanner: A signature-matching module that uses hex and string patterns to detect malware, shellcode, and
     steganography tool footprints.
   * Cracker: A dictionary-based brute-force module that uses a wordlist.txt to unlock encrypted PDFs, ZIPs, and
     hidden Steghide payloads.

  3. Frontend (Web Interface)
  The frontend is a Single Page Application (SPA) designed to visualize complex forensic data.


   * React (JavaScript): The UI is built using React components, providing a smooth, interactive experience for      
     dragging-and-dropping files.
   * Recharts: A charting library used to generate the Entropy Maps. This allows you to visually "see" encrypted data
     as high-entropy spikes in a file's structure.
   * Vanilla CSS3: Used for styling the dark-themed, professional dashboard without the bloat of heavy CSS
     frameworks.
   * HTML5 / Canvas: Used for rendering previews of images and binary data.


  4. External Dependencies
   * Steghide (Optional): If installed on your system, the tool automatically integrates with it to extract data
     hidden using traditional steganographic methods.

  1. Why Use This Project?
   * Malware Detection: Modern malware often hides its configuration or payloads (like cmd.exe or powershell
     commands) inside innocent-looking images or PDF files to bypass firewalls and antivirus software.
     StegDefenderPro unmasks these "droppers."
   * CTF (Capture The Flag) Tool: It is a powerful tool for cybersecurity competitions, automating the process of
     extracting hidden "flags" from PCAPs, images, and documents.
   * Data Leakage Prevention: Companies can use it to scan outgoing attachments to ensure employees aren't hiding
     sensitive documents inside images or audio files (Insider Threat Detection).
   * Protocol Integrity: It verifies that your network traffic (like DNS and Ping) isn't being used as a "covert
     channel" for data exfiltration.

  2. What Makes This Project Different?
  Most steganography tools are "single-shot"—they look for one specific thing. StegDefenderPro is different because  
  of these 4 unique features:


  A. The Recursive Engine (Deep Scan)
  If a hacker hides an encrypted ZIP inside a PDF, and inside that ZIP is an image with an LSB-hidden message:
   * Standard Tool: Might find the ZIP and stop.
   * StegDefenderPro: Extracts the ZIP $\rightarrow$ Extracts the image $\rightarrow$ Extracts the message. It builds
     a Forensic Tree of every hidden layer it finds.


  B. Protocol Steganography Detection
  Most tools focus only on files. This project analyzes the Network Protocol itself. It detects if data is hidden
  inside:
   * ICMP (Ping): Extra bytes in the "Echo Request" payload.
   * DNS Tunneling: Encoded data in subdomains (e.g., a1b2c3d4.example.com).


  C. Automated Cracking & Decryption
  It doesn't just find encrypted data; it tries to solve it. It has a built-in Cracker and CryptoManager that:       
   * Automatically uses a wordlist to brute-force PDF/ZIP passwords.
   * Automatically detects AES-GCM or ChaCha20 encrypted strings and tries to decrypt them on-the-fly.


  D. Visual Forensic Analysis (Entropy Mapping)
  It uses Entropy Analysis to "see" what the human eye cannot.
   * How it works: Encrypted or compressed data has very high "entropy" (randomness).
   * The Difference: The tool generates a visual chart of the file. If you see a sudden spike of high entropy in the
     middle of a "clean" image, you know exactly where the hidden data starts.

   In the world of cybersecurity, there are two sides:
   1. Steganography: The art of hiding a secret (e.g., Steghide, OpenStego).
   2. Steganalysis: The art of detecting and extracting those secrets.

  StegDefenderPro is on the Steganalysis (Defense) side.


  Here is how to think about it:
  If a "steganography tool" is like a locked safe used to hide a document, StegDefenderPro is the X-ray machine and
  the locksmith used to find the safe and open it.


  What it DOES (The "Detective" Work):
   * X-Rays Files: It looks deep inside the binary code of images, PDFs, and ZIPs to find "unnatural" patterns.
   * Extracts Artifacts: It pulls hidden files out of other files (Recursive Extraction).
   * Cracks Passwords: It uses its brute-force engine to guess the passwords of hidden data.
   * Sniffs Networks: It monitors network traffic (PCAPs) to see if data is being smuggled through pings or DNS
     queries.
   * Analyzes Entropy: It calculates the mathematical randomness of a file to "see" where encryption or compression  
     is hiding.


  What it does NOT do:
   * It does NOT hide data: It won't help you put a secret message into a cat picture.
   * It does NOT encrypt for the purpose of hiding: It only encrypts/decrypts to test security and forensic
     capability.


  In short:
  StegDefenderPro is a Forensic Defense Tool. It is used by security experts to catch hackers, spies, or employees   
  who are using steganography to steal data or hide malware.
   General & Architecture
   1. Project Overview: It is a Recursive Forensic Engine. Unlike simple tools that check a file once,
      StegDefenderPro extracts hidden artifacts and recursively analyzes them. If a ZIP is hidden in an image, it    
      extracts the ZIP and then analyzes the files inside that ZIP for further steganography.
   2. Architecture: It follows a modular architecture. The core/engine.py acts as the orchestrator. By separating    
      core (logic/orchestration) from analyzers (type-specific detection), the system is easily extensible—you can   
      add support for a new file format just by creating a new analyzer class.
   3. Workflow:
       * Identification: FileIdentifier uses magic bytes to find the true file type.
       * Selection: The Engine selects the appropriate Analyzer.
       * Analysis: The analyzer performs structural checks and extracts hidden artifacts.
       * Global Scan: The Engine runs YARA rules and VirusTotal checks on the file.
       * Recursion: Any extracted artifacts are fed back into the engine for analysis.
       * Reporting: Results are aggregated into a PDF report.
   4. File Identification: The file_identifier.py uses the filetype library (magic bytes) but includes manual header 
      overrides for specialized formats like PCAP (\xd4\xc3\xb2\xa1) which generic libraries might miss. It falls    
      back to file extensions only if magic bytes fail.


  Core Steganography Concepts
   5. Entropy Analysis: Entropy measures the "randomness" of data. In entropy.py, a Shannon entropy score close to
      8.0 indicates that the data is highly random. Since encrypted or compressed hidden payloads look like random
      noise, a high entropy score in a normally structured file is a major "red flag."
   6. LSB vs. EOF:
       * EOF (End of File): The tool looks for data appended after the standard file terminator (e.g., FF D9 for
         JPEGs or %%EOF for PDFs).
       * LSB (Least Significant Bit): In image_analyzer.py, the tool extracts the lowest bit of every pixel and
         calculates its entropy. If the LSB plane has high entropy (>7.8), it likely contains an encrypted hidden
         message.
   7. Extraction Process: When an analyzer identifies hidden data (like an appended ZIP), it uses save_artifact().
      This writes the binary data to a temporary directory. The ForensicEngine monitors this directory and
      automatically triggers a new analysis pass on the newly created file.
   8. Polyglots: These are files that are valid in two different formats (e.g., a GIF that is also a ZIP).
      StegDefenderPro handles this by checking for internal signatures. For example, the ArchiveAnalyzer checks if a
      file contains a ZIP "End of Central Directory" record, even if the file header says it's an image.


  Specialized Analyzers
   9. Network Steganography: The network_analyzer.py uses scapy to:
       * Check for high-entropy data in ICMP Echo Request (Ping) payloads.
       * Detect DNS Tunneling by looking for unusually long or random subdomains in DNS queries.
       * Reassemble TCP streams to extract files being transferred over HTTP.
   10. Text Steganography: It detects Zero-Width Characters (ZWC) such as U+200B (Zero Width Space) and U+200C
       (Non-Joiner). These characters are invisible to the human eye but can be used to encode binary data (0s and
       1s) between visible letters.
   11. PDF Analysis: The pdf_analyzer.py uses pypdf to find:
       * Suspicious Keys: /OpenAction or /JavaScript which can trigger malicious code.
       * EmbeddedFiles: Hidden attachments within the PDF structure.
       * Appended Data: Content added after the final %%EOF marker.
   12. Audio Analysis: For .wav files, it checks the RIFF header size against the actual file size. If the file is
       larger than the header declares, the extra bytes are extracted as a hidden payload. For .mp3, it looks for
       multiple ID3 tags, which suggests file concatenation.


  Security & Advanced Features
   13. YARA Scanning: yara_scanner.py uses signature matching to find known patterns in the hidden data, such as
       Shellcode (NOP Sleds), PowerShell commands, or signatures of other steganography tools like Steghide and
       OpenStego.
   14. VirusTotal Integration: vt_scanner.py calculates the file's SHA256 hash and checks it against a database of
       known malware. This is vital because steganography is often used as a "dropper" for malware; the file itself
       might look like an image, but the hidden payload is a virus.
   15. The Cracker Module: The cracker.py performs dictionary attacks using wordlist.txt. It supports:
       * ZIP/PDF: Attempting to decrypt password-protected archives/documents.
       * Steghide: Using the steghide info command to test passwords against images.
   16. Metadata Extraction: It extracts EXIF data (images) and Document properties (PDF). This is important because
       steganography tools often leave "fingerprints" in metadata (e.g., the Software tag might change), or users
       might hide short text strings directly in metadata fields like Author or Comments.


  Implementation & Tools
   17. Tech Stack:
       * Python: Used for its powerful forensic libraries (scapy, PIL, pypdf).
       * FastAPI/Flask (Web UI): Allows for a user-friendly way to upload files and visualize complex data like
         entropy maps which would be hard to read in a CLI.
   18. Error Handling: The engine uses try-except blocks for each analyzer. If a file is "locked" or corrupted, the
       tool doesn't crash; instead, it reports a "Warning" in the details and suggests using the Cracker module.
   19. Reporting: pdf_reporter.py uses the fpdf library. It color-codes findings based on severity: DANGER (Red) for
       confirmed stego, WARNING (Orange) for suspicious anomalies, and INFO (Black) for general file properties.
   20. Scalability: To add a new file type (e.g., Video):
       * Create video_analyzer.py in the analyzers/ folder inheriting from BaseAnalyzer.
       * Register the Video MIME type in core/engine.py's _get_analyzer method.
       * The rest of the engine (YARA, VT, Entropy, Reporting) will automatically work with the new analyzer.
